## Mandatory Renaissance Visual Direction

This portfolio must not look like a generic modern SaaS portfolio.

The design must clearly communicate a Renaissance-inspired identity through visible UI decisions, not only text.

Claude must actively inspect the current UI/code and decide what visual elements are missing.

Required Renaissance-inspired improvements may include:
- parchment or antique paper background treatment
- subtle canvas/noise texture
- antique gold borders
- classical framed project cards
- ornamental section dividers
- museum/gallery-style project presentation
- manuscript-inspired content panels
- elegant serif display typography for headings
- warm ivory, deep brown, burgundy, muted gold, olive, and charcoal palette
- decorative corner accents
- curated art-gallery layout
- classical hover interactions

Avoid:
- only editing text
- only changing copywriting
- generic gradient cards
- generic SaaS layout
- neon/futuristic/cyberpunk style
- overusing glassmorphism
- adding random decoration that hurts readability

When improving the portfolio, Claude must:
1. Audit the current visual state first.
2. Explain why the current design does or does not feel Renaissance.
3. Identify specific missing visual elements.
4. Implement visible UI changes across multiple relevant sections.
5. Preserve working functionality.
6. Check responsive behavior.
7. Run build/lint if available.
